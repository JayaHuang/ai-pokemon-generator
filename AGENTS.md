## 🧠 数字员工记忆系统

> 记忆系统文档: `.codex/memory/MEMORY_SYSTEM.md`
> Agent 记忆模板: `.codex/memory/templates/agent-memory-template.md`
> 迭代协议: `.codex/memory/templates/iteration-protocol.md`

### 记忆系统简介

每个数字员工在执行任务时，会自动记录"记忆"——包括执行日志、用户反馈和可复用的经验模式。这些记忆会被后续调用时读取，帮助数字员工自我迭代优化。

**核心原则**: 记忆驱动迭代 = 经验沉淀 × 反馈循环 × 模式优化

### 记忆目录结构

```
projects/ai-pokemon-generator/.codex/memory/
├── feedback/          # 用户反馈、修正记录、质量评分
├── logs/              # 任务执行日志、决策记录
└── patterns/          # 提取的成功模式、最佳实践
```

### 记忆使用流程

#### 1. 读取记忆 (READ)

当数字员工被调用时，会自动：
1. 读取 `AGENTS.md` + `PROJECT_SPEC.md` (已有行为)
2. 读取 `.codex/memory/patterns/` 中的相关模式
3. 读取 `.codex/memory/feedback/` 中的最近3条反馈
4. 读取 `.codex/memory/logs/` 中的最近1条同类日志

#### 2. 执行任务 (EXECUTE)

执行任务时，会：
- 在关键决策点参考历史模式
- 应用上次反馈中的改进建议
- 主动避免之前出现过的问题

#### 3. 写入记忆 (WRITE)

任务完成后，会自动：
1. 在 `.codex/memory/logs/` 写入任务日志
2. 询问用户对本次输出的评价
3. 如果有反馈，在 `.codex/memory/feedback/` 写入反馈记录
4. 如果发现可复用的经验，更新或创建模式文件

### 迭代频率

| 迭代类型 | 频率 | 负责人 |
|---------|------|--------|
| 任务级迭代 | 每次任务 | Agent 自身 |
| 周度回顾 | 每周 | 项目经理 |
| 月度优化 | 每月 | 项目经理 |
| 季度复盘 | 每季度 | 用户 + 项目经理 |

### 记忆系统健康度指标

- **记忆覆盖率**: 有记忆记录的任务数 / 总任务数 (目标 ≥ 80%)
- **反馈响应率**: 有反馈的任务数 / 总任务数 (目标 ≥ 60%)
- **模式复用率**: 应用了模式的任务数 / 总任务数 (目标 ≥ 40%)
- **质量提升率**: (新任务评分 - 旧任务评分) / 旧任务评分 (目标 ≥ 10%)
